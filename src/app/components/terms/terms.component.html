Terms

  • What is Angular?

    A Component Based Framework built using JavaSscript, that allows for the development of scalable applications.

  • What are the technologies used in Angular?
    1. NPM (Node Package Manager)
    2. Angular CLI (Command Line Interface)
    3. TypeScript
    4. HTML
    5. CSS
    6. RXJS (Reactive Extensions for JavaScript). In Angular used as a library to Subscribe to a service using Observables
  • How does an Angular application work?
    1. Download and install NodeJs
    2. Use the terminal command npm i -g
    3. Use the terminal command npm i @angular/cli
    4. Navigate to designated folder for project
    5. Run the command ng new <name of application>
    6. run command npm i to install node_modules
    7. npm start or ng serve to start the application
    8. When an Angular app runs, the first file read is the main.ts file. This is where it bootstraps the AppComponent Class
    9. Angular then looks for the <app-root> which is located in the index.html file
    10. Angular creates, updates, and destroys components as the user moves through the application. These are called lifecycle Hooks.
  • What are custom elements?

    Can be used to create reusable components or define new and existing HTML tags

  • Directives
    • Can be built in or custom
    • An example of a built in directive is ngClass>
    • Directives are Classes that can add or modify behavior within a template
  • Observables
    • Observable pass messages between different parts of an application asynchronously which means, multiple pieces of data can be called at once until the component unsubscribes
    • Observables use the RXJS Library
  • Promises
    • Promises deal with only one asynchronous event at a time and were created to prevent callback hell. You can run into the same issue by using a bunch a thenables though so, await and async allows you to control how the behavior is fired off.
  • Lazy Loading
    • Lazy Loading helps with the performace of an application by only loading the necessary parts of an application as they are requested by the user
    • Lazy loading splits the application into a bunch of bundles.

      • Dependency Injection

      Allows components, decorators, classes, pipes, and injectables to configure the dependencies they need

      An example would be using a service by giving it a name within a constructor and injecting the dependency by using important within the component where it is being used

  • Session Storage vs Local Storage
    • Session Storage - Data expires once a user leaves or closes the browser
    • Local Storage - Data is collected and saved and doesn't expire
  • HTML Improvements
    • Checks for titles and metadata descriptions
  • Speed Up an Application
    1. Reduce the number of HTTP Requests. If you have already made a request, the data should already be in your application. Make sure you are not requesting more than once
    2. Optimize Images
    3. Write Mobile first code
    4. Use AOT(ahead of time) Compile during the build phase
    5. Use Pure Pipes. Only customize when necessary
    6. Use trackBy-a directive that can work with ngFor to identify items in a list of DOM elements)
  • Security Best Practices for UI Development
    • Include everyone in the practices. Everyone needs to be aware of the vulnerabilities within an application
    • Use some sort of Cyber Security such as HIPAA or FISMA
    • Automate and Integrate security tools such as CI/CD platforms and other issue trackers
    • Follow Secure software development practices – how to prevent SQL injections
    • Diverse security measures – Firewall, Scanners
    • SONAR QUBE
  • Cross-site request forgery
    • An unintential carry out by the user such as a request to change their password or email
  • Route Guards

      Check conditions before a user is allowed to navigate to certain parts of the app

    • Behavior Subject - Holds one value that will return that value or a current value upon subscription.

    • Subject - Does not hold a value and does not return a value upon subscription using Next()
