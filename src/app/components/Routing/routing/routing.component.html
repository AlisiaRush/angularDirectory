Query Parameters vs. Route Parameters

Query Parameters

  1. Query Parameter- Gets specified at the end of the URL separated by "?" They are not required and are not part of the route definition. If you don't specify a value for the query, it will still navigate to the URL.
  2. Route Parameter- Required to determine the route definition. They are required

Fragments

A Fragment in URL is a link which jumps to a section or a content HTML page which contains the ID mentioned in the fragment. A fragment comes after the "#" sign

Example: localhost:4200/Products#service

We should be able to select a navigation link and anchor down to that component in the same page.

Route Guards

Used to control whether the user can navigate ot or away from the given route based on some condition.

Use Cases:

  1. To confirm the navigation operation
  2. Asking whether to save data before moving away from a view
  3. Allow access to certain parts of the application to specific users
  4. Validating the Route parameters before navigating to the route
  5. Fetchin some data before you display the compnent view

Types of Route Guards

  1. CanActivate - Decides if a route can be activated or not. This Route Guard can be used in cases where the user is not authorized to navigate to the target route.
  2. CanDeactivate
  3. Resolve
  4. CanLoad
  5. CanActivateChild
