HTTP Requests

HTTP Requests - Makes calls to the server where the data is present. We don't use credentials to connect directly to a data base in the Angular App! SECURITY!!! We will use an API

We connect and communicate with an API's

URL's - domain path such as (yourdomain.com/products/id)

HEADERS - Metadata for the request

BODY - Data sent to the server.

HTTP Verbs

  1. GET - Gets data from server
  2. POST - Posts data to server
  3. PUT - Updates data to server
  4. PATCH - A set of instructions on how to modify a resource. Similar to "UPDATE". The difference is... Patches minimize your attack surface and protects against attackers. Patches address specific vunerabilities.
  5. DELETE - Deletes data from server

Setting up FireBase

  1. Go to Firebase website: https://firebase.google.com/
  2. Click on "Get Started"
  3. Create a Project and provide a name
  4. Accept and continue
  5. Create Project
  6. Click on Continue
  7. Select Build option in left panel
  8. Create a database > Create realtime database
  9. Select Location and click on Next
  10. Select start in test mode. Authentication will be added later.
  11. You should see an endpoint in the realtime database
